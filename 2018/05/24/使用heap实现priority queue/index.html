使用heap实现priority queue

2018-05-24
cs61b

priority queue（优先队列）

在优先队列中，元素被赋予优先级，当访问元素时，具有最高级优先级的元素先被访问。

Heap

实现优先队列（priority queue）有很多种方法，可以使用排序的数组来实现，可以使用二叉搜索树来实现，但都不是最优的实现方式。使用二叉搜索树（BST）无法处理相同节点，并且还要不断保持二叉树的平衡。故我们使用堆结构（heap）来实现优先队列。

二叉堆结构，基本就是二叉树，但是还要满足两个条件：

  • 满二叉树（ the tree must be complete）
  • 每一个节点要小于它的左右孩子（every node is smaller than its descendants）

满二叉树是指，只有叶子层可以不是满的，但叶子必须从左到右依次排列。

由堆结构特点，用它来实现优先队列是非常自然的。要返回最小的元素，只需返回根节点即可。

堆结构最基本的两个操作：添加元素，删除元素

添加元素

为了不破坏堆结构的特性，我们将元素添加至叶子层最左边可以添加的位置。然后不断将元素与它的父节点相比较，如果小于父节点，则互换，直到它比父节点大或者它成为了根节点。这一过程称作bubbling up或swimming。

删除元素

我们要删除根节点，不能直接删除，而是要找到根节点的替换节点。所以我们先将根节点与叶子层最右边的叶子互换，删掉最右边的叶子。然后不断将新的根节点与它的左右孩子比较，如果小于左右孩子中较小的那一个，则将节点与较小的孩子节点互换，直到它比左右孩子节点都大。这一过程称作sinking。

heap的表示

我们可以使用Linked Node来表示heap，也可以采用Array来表示。由于heap是满二叉树，所以这里我们采用Array来表示heap，比较简单。

将 index = 0 的位置置空，将根节点放在 index = 1 的位置上，然后依次放入1的左右孩子2，3 。再依次放入2，3的左右孩子……这样，在访问某一个索引为k的节点的孩子和父节点时，就有一个简单的数量关系：

  • leftChild(k) = k*2
  • rightChild(k) = k*2 + 1
  • parent(k) = k/2

时间复杂度分析

heap的添加操作add： O（logN）

heap的删除操作removeSmallest： O（logN）

heap的查找操作getSmallest： O（1）
