ArrayRingBuffer实现及写Iterator

2018-05-03
cs61b

接口：BoundedQueue

BoundedQueue有点类似Deque，但是仅提供较少的API。 item只能从队列后添加，从队列头出队。而且不像Deque，BoundedQueue容量固定，当队列满的时候，不能继续添加item。

抽象类：AbstractBoundedQueue

类可以实现接口，继承接口的所有方法和变量。如果该类没有实现所有的接口的方法，那么该类必须声明为抽象类。不像接口，抽象类可以提供方法实现的继承和成员变量的继承。

这个抽象类只是为了提供两个protected成员变量，以及两个getter方法。这些变量和方法可以被子类继承，从而节省了子类（ArrayRingBuffer）的实现工作。一点点：）

实现：ArrayRingBuffer

我们使用Array来实现BoundedQueue。一个naive的实现方式是：在数组中顺序插入新的item，出队时将数组元素一次向前挪一位。很明显，这是一种非常低效的实现方式。

我们可以使用ring buffer这种数据结构来实现，有点类似于循环数组。设有first和last两个指针，first指向最先被插入的元素，即队列头；last指向下一个可以插入数据的index。插入数据时，将数据插入last的位置，然后增加last。删除数据时，将first指向的数据删除，然后增加first。当 index == capacity 时，设置index = 0 。

Iterator

我们可以看到，BoundedQueue接口继承了Iterable，说明这个接口是支持迭代的，也定义了生成迭代器的抽象方法。故而我们需要在实现类中实现这个方法。

这个iterator方法，会返回一个迭代器。所以我们在ArrayRingBuffer类中声明一个私有的nested class。

这个bufferIterator要实现Iterator接口，所以要实现两个方法：hasNext() 和 next() 。判断是否迭代完成以及返回要迭代的item。

完成之后，我们的这个ArrayRingBuffer就具有了生成迭代器的方法，可以使用iterator()来调用，生成一个迭代器。同时，也可以使用foreach循环来迭代。（推荐）
