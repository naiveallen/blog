Git+Hexo初体验

2017-12-29
github, hexo

前几日学习了一下Git的使用，虽浅尝辄止，却也似发现了新大陆。Git异常强大的功能加上github提供的服务，便萌生了搭建自己的个人博客的想法。本篇仅记录一下这一两天遇到的问题，具体教程请参阅HEXO+Github,搭建属于自己的博客。

npm安装

首先遇到的问题就是让人头疼的npm安装。众所周知，国内环境下使用npm下载所需要的package非常非常非常慢，但是我们中国人民是不信邪的！于是尝试着使用npm下载了一下，十分钟都没有动静……放弃了……╮(╯﹏╰）╭于是乎，我又开始安装淘宝的国内npm镜像，指令如下：

十分钟过去了，仍然没有动静……(｡•́︿•̀｡)我开始怀疑我的gitbash有问题，遂换到Windows的cmd，仍然没反应。忍无可忍，我掏出了终极大杀器——科学上网， 换到了法国的IP，终于，成了！在cmd里把淘宝npm镜像装好了。安装好之后，用cnpm命令安装也就快了许多。

Theme语言更换

打开/blog的目录下的_config.yml ，将language参数设置为zh-CN 。

Git add命令

修改了theme文件后，我以为要使用git add，commit，push等命令将文件上传到repository中去，奈何使用git add一直添加不成功，也没有报错，甚是费解。查了许多资料，说git add不能添加空的文件夹。于是我进repository中看了一眼，(ｷ｀ﾟДﾟ´)!! 我的theme文件夹是空的！那我的theme样式怎么部署到github上的…… 不过正确部署后，本地的东西都可以部署到github上，空文件夹暂且留作一个未解之谜，今后理解了再来更新。

部署

使用hexo deploy部署了几次，发现有些时候所做的更改未能成功上传。遂查资料：

Why aren’t my changes deployed to the Github Pages?

Please run these commands before hexo deploy:

$ hexo clean
$ rm -r .deploy_git

运行上面两行命令后，部署就没有什么大问题了。

搞了一天，终于把博客搭建好了，还有很多未填的坑，还在摸索中，等慢慢的完善好了，挑时间整理一下在这里更新。总之，个人博客已搭建完成，开始CS61B的学习。(｡◕ˇ∀ˇ◕)
